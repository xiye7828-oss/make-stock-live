# rewrite-patterns.md

> 改写模式参考手册
> 用途：polish / rewrite 模式处理各类原稿的标准路径
> 每种模式包含：典型原稿问题、必删内容、必留内容、改写方向、输出气质、示例说明

---

# 模式一：策略稿降级为知识直播稿

## 典型原稿问题

- 大量具体点位：“4060支撑”“4100压力”
- 具体仓位建议：“七成仓”“八成仓”
- 具体操作动作：“加仓”“减仓”“低吸”
- 次日走势预判：“明天高开”“明日看跌”
- 板块具体标的：“买券商”“买银行”
- 个股代码和名称
- 互动索取：“扣1扣2”“飘个6”
- 礼物感谢段落
- 盘中数据播报过多，且没有转化为分析框架

## 必删内容

1. 所有数字点位（替换为“均线支撑”“关键位置”“整数关口”）
2. 所有仓位比例（替换为“仓位管理原则”“分批原则”）
3. 所有加减仓动作（替换为“判断框架”“应对原则”）
4. 所有次日预判（替换为“观察条件”“判断信号”）
5. 所有具体板块标的（替换为“板块轮动逻辑”“风险类型”）
6. 所有个股代码和名称
7. 礼物感谢段落（默认删除，必要时仅保留一次且极简）
8. 互动索取口号（改为“记住了吗”“这个点先记一下”类回拉确认）

## 必留内容

1. 市场判断逻辑（涨/跌的本质原因分析）
2. 风险识别框架（哪种情况要小心）
3. 情绪误区捕捉（常见错误心理）
4. 观察框架（怎么分析、怎么判断）
5. 板块轮动规律（不涉及具体标的）
6. 真假强弱识别方法

## 正确改写方向

**原文 → 改写后：**

- “今天跌到4062支撑住了” → “今天打到关键均线位置获得支撑”
- “七成仓操作” → “分批建仓原则”
- “明天要低吸” → “观察条件满足时可以关注”
- “券商要涨” → “券商板块的观察逻辑是……”
- “某只票涨了5%” → “相关板块近期表现偏强”
- “扣1扣2” → “这个框架记住了吗”
- “感谢老铁送的礼物” → 整段删除

**核心原则：**  
把“操作建议”替换为“判断框架”，把“具体动作”替换为“观察条件”。

## 输出气质

- 像老师在讲判断方法，不是在下操作指令
- 讲“为什么”，不讲“做什么”
- 给框架，不给答案
- 引导思考，不引导行动

## 示例说明

**原稿片段：**
> 今天大盘跌到4060支撑位，我告诉大家支撑是4050-4060区间，跌到这里就有支撑。今天缩量1700亿，说明主力在减仓。明天要小心，建议减仓到七成。

**改写后：**
> 今天打到均线支撑位置获得企稳。这个位置能不能撑住，要看两个信号：第一，卖盘有没有缩；第二，有没有资金进场。缩量说明多空都在观望，不是好事。这种时候怎么判断？关键是看涨的时候带不带量。带量涨是真强，缩量涨是虚的。这个框架你先记一下。

# 模式二：文章腔改成口播稿

## 典型原稿问题

- 书面语多：“因此”“然而”“由此可见”
- 句式完整复杂：从句套从句，一句话说三件事
- 缺乏停顿感：没有呼吸空间，一口气念不完
- 没有互动信号：不问观众、不确认理解
- 逻辑连接靠书面词：“首先、其次、最后”
- 缺乏情绪起伏：全程同一个节奏
- 开头太正：上来就讲正经话题，不抓人

## 必删内容

1. 连接词：“因此”“然而”“由此可见”“综上所述”
2. 复杂从句结构（拆成短句）
3. 过多并列要点（超过5个的压缩到3个）
4. 理论化表述（不落地的分析）
5. 开头铺垫过长（直接入题）

## 必留内容

1. 核心观点（压缩成一句能记住的话）
2. 判断框架（方法论部分）
3. 具体例子（用来解释框架）
4. 风险提醒（必须有结尾）
5. 节奏停顿感（换气口）

## 正确改写方向

**原文 → 改写后：**

- “因此我们可以得出结论” → “所以这一条你记住：”
- “然而这种情况并非常态” → “但有的时候不是这样”
- “由此可见” → 删除，改为直接说结论
- “首先、其次、再次、最后” → 改为“第一……第二……第三……”，超过3点就压缩
- “这说明了一个什么问题呢” → “说明什么？说明……”
- “从历史数据来看” → “以前也出现过这种情况”

**核心原则：**  
把“写出来的文章”改成“讲出来的话”。念出来不顺的，全部改。

## 输出气质

- 一口气能念完
- 短句为主
- 有停顿、有反问
- 像有人在跟你说话，不是在读报告

## 示例说明

**原稿片段：**
> 从技术分析的角度来看，大盘的成交量是判断市场强弱的重要指标。当价格上涨时如果成交量同步放大，则说明上涨动力充足，反之则表明上涨动力不足。因此，投资者应当密切关注成交量的变化，并将其作为判断市场趋势的重要参考依据。

**改写后：**
> 判断大盘强不强，有一个最简单的方法：看涨的时候带不带量。  
> 涨的时候放量，是真强。涨的时候缩量，是虚的。  
> 为什么？因为有人愿意花钱往上推，才是真的强。缩量涨，只是有人在维护图形，不是有资金主动做多。  
> 所以你记住：涨的时候看量，量是关键。

# 模式三：低质量直播话术改成知识型直播话术

## 典型原稿问题

- 无意义重复：“来了来了兄弟们”“家人们谁懂啊”
- 情绪口号化：“太牛了”“绝了”“太给力了”
- 空洞感叹：“这个市场啊”“不得不说”
- 重复互动索取：“扣666”“飘起来”
- 信息密度为零的废话
- 用感叹词代替分析
- 缺乏实质内容，只有情绪渲染

## 必删内容

1. 无意义重复开场（“来了来了兄弟们”）
2. 空洞感叹词（“不得不说”“真的是”）
3. 情绪口号（“太牛了”“绝了”）
4. 重复互动索取（超过3次的“扣1扣2”）
5. 信息密度为零的废话段落
6. 纯情绪渲染没有分析的段落
7. “家人们”类饭圈化表达

## 必留内容

1. 真实分析框架
2. 具体判断方法
3. 有信息量的观察角度
4. 有效互动（确认理解类）
5. 关键记忆句
6. 风险提醒

## 正确改写方向

**原文 → 改写后：**

- “兄弟们来了来了，不得不说今天这个盘面啊，真的是” → “好，今天大盘这个走法”
- “太牛了家人们，这个涨得太给力了” → “这个板块今天表现偏强，但我们看……”
- “扣666扣起来” → “这个点你先记一下”
- “这个市场怎么说呢，你懂的” → 删除，直接说观点
- “不得不说这个主力啊，它真的” → “主力在做什么？我们来看……”

**核心原则：**  
保留直播感，去掉空话。把“渲染情绪”改成“传递信息”。

## 输出气质

- 有信息量
- 有分析框架
- 有记忆点
- 有节奏但不做作
- 像一个有内容的老师，不像一个喊麦的

## 示例说明

**原稿片段：**
> 来了来了兄弟们，今天这个大盘真的是，绝了，你看这个走势，谁懂啊，太牛了。扣666扣起来，家人们，把小红心给我点起来。不得不说这个主力啊，它这个操盘手法，真的是太厉害了。

**改写后：**
> 好，今天大盘这个走法，简单说一下。  
> 表面看是涨的，但你要会看。黄线弱、白线强，说明涨的是少数权重股，不是全面做多。  
> 这种情况通常意味着什么？意味着行情可能快到阶段性高点了。主力用少数股票撑指数，掩护别的股票出货。  
> 这是个判断框架，你记住：白线在上、黄线在下，不是好信号。反过来才说明真强。  
> 这一层先记一下。

# 模式四：不贴老师风格改成贴老师风格

## 典型原稿问题

- 风格和老师人设不匹配
- 老师是理性拆解型但稿子写得像陪伴答疑型
- 老师擅长框架教学但稿子里全是情绪互动
- 开头风格和整体风格不一致
- 没有按老师习惯的句式组织语言
- 和老师平时的表达方式落差太大

## 必删内容

1. 不符合老师人设的表达方式
2. 老师不擅长的内容类型表达
3. 老师不用的口头禅和句式
4. 和老师日常风格差异过大的段落
5. 老师说不出口的话

## 必留内容

1. 符合老师风格的核心框架
2. 老师擅长讲的内容
3. 老师习惯用的分析角度
4. 老师风格的记忆点句式
5. 老师习惯的节奏安排

## 正确改写方向

**理性拆解型老师：**
- 多用“第一步、第二步、第三步”结构
- 讲原因、讲逻辑、讲底层
- 少用感叹句
- 少情绪渲染
- 多框架输出
- 收尾压记忆句

**老师带教型老师：**
- 多用“来、跟我走、记住这个”句式
- 预设观众可能不懂的地方并提前解答
- 语速中慢，多重复重点
- 结尾有“今天带走了什么”感
- 适合框架启蒙类内容

**陪伴答疑型老师：**
- 多用“你可能也有这种感觉”句式
- 主动说出观众心里在想什么
- 语气偏暖，像朋友聊天
- 结尾留接口让观众继续问
- 适合心态类、焦虑类话题

**强观点提醒型老师：**
- 开头先抛反常识观点
- 短句为主，干脆利落
- 少解释多结论
- 适合留人开场和风险提醒
- 结尾给明确判断句

## 输出气质

- 像这个老师平时在讲的
- 观众听了不会觉得人格分裂
- 老师自己能讲顺
- 符合老师日常表达的节奏

## 示例说明

**理性拆解型老师改写：**

**原稿（偏陪伴型）：**
> 兄弟们，你们有没有这种感觉，买了就跌，卖了就涨，我太懂这种感觉了。今天我们来聊聊这个问题。

**改写后（理性拆解型）：**
> 今天讲一个误区：为什么你总在错误的位置买卖。  
> 这个问题表面看是运气问题，但拆开看，是判断框架问题。  
> 我分三步讲：第一步，什么叫“错误的位置”。第二步，为什么会判断错。第三步，怎么建立正确的判断框架。

**老师带教型老师改写：**

**原稿（偏理性型）：**
> 判断大盘强弱的指标有三个：量、价格、情绪。量化指标体系是……

**改写后（老师带教型）：**
> 好，今天教你一个判断大盘强弱的框架，你学会了，自己能判断，不用问别人。  
> 来，先记住三个东西：量、价格、位置。这三个怎么看，我一步一步讲，你先跟上我的思路。  
> 第一步，看什么？先看量……

# 通用改写检查清单

## 每篇稿子过一遍

- [ ] 有没有具体点位数字 → 替换为框架描述
- [ ] 有没有仓位建议 → 替换为管理原则
- [ ] 有没有加减仓动作 → 替换为观察条件
- [ ] 有没有次日预判 → 删除或替换为判断框架
- [ ] 有没有具体板块标的 → 替换为板块逻辑
- [ ] 有没有个股代码名称 → 删除
- [ ] 有没有空洞感叹词 → 删除
- [ ] 有没有念不顺的句子 → 改短
- [ ] 符不符合老师风格 → 按风格调整
- [ ] 有没有记忆句 → 没有就加

## 输出前的最终确认

1. 这份稿子是“讲判断框架”，不是“下操作指令”
2. 观众听完能自己判断，不需要问下一步
3. 老师能直接念出来，不需要改口
4. 符合股票知识直播的合规要求

# 成功样例沉淀

## 样例1：盘面策略稿降级为股票知识直播稿

### 原稿问题
- 大量具体点位
- 大量仓位比例
- 明确加仓、减仓、低吸动作
- 明确次日走势预判
- 板块和个股跟随建议过多
- 互动方式偏低质量直播
- 整体更像“带操作”，不像“讲框架”

### 改写目标
把原稿从“策略型直播口播”改成“股票知识直播口播”。

### 改写原则
1. 删除具体点位、仓位、加减仓、低吸、止盈、回避等执行口径
2. 删除“明天怎么走”“明天怎么应对”类直接策略判断
3. 删除具体板块跟随建议和个股应对建议
4. 保留市场判断逻辑、风险识别思路、情绪误区、观察框架
5. 语言要像老师在直播间讲判断框架，不像直播间带操作

### 改写后应保留的核心内容
- 大盘强弱判断框架
- 真强假强识别方法
- 诱多陷阱识别
- 板块轮动观察逻辑
- 外围联动分析框架
- 风险识别框架
- 持仓心态框架
- 总结记忆点

### 改写后不应出现的内容
- 4060 / 4100 这类具体点位
- 七成仓 / 八成仓这类比例
- 加仓 / 减仓 / 低吸 / 高抛
- 明天高开 / 低开 / 变盘
- 具体板块机会建议
- 具体个股应对口径

### 输出气质
- 更像知识直播
- 更像老师讲判断方法
- 更适合长期复用
- 更稳、更克制
- 有直播感，但不低质

### 成功判断标准
- 策略属性明显下降
- 知识框架明显增强
- 没有具体操作建议
- 老师可以直接讲
- 总结能压成可复用框架

### 备注
遇到同类原稿时，优先按本模式处理：
先做知识直播降级改写，再做口播优化，不直接在策略稿上润色增强。

## 样例2：强策略复盘稿改写为知识直播框架稿

### 原稿问题
- 大量“明天怎么走”的判断
- 大量支撑位、压力位、点位推演
- 大量仓位控制、加减仓、低吸表达
- 板块和个股应对建议过多
- 互动方式偏低质量直播
- 整体更像“盘后策略复盘”，不像“知识直播讲框架”

### 改写目标
把原稿从“强策略复盘稿”改成“股票知识直播框架稿”。

### 改写原则
1. 删除具体点位、仓位、加减仓、低吸、减仓、回避等执行型口径
2. 删除明日走势预判和次日应对方案
3. 删除具体板块跟随建议和具体个股应对建议
4. 保留大盘判断逻辑、板块轮动逻辑、外围联动逻辑、风险识别框架、心态误区
5. 最终输出必须更像老师在讲判断框架，而不是在带明天怎么做

### 改写后应保留的核心内容
- 大盘强弱判断框架
- 真强假强识别
- 诱多陷阱识别
- 板块轮动分析框架
- 外围市场联动分析框架
- 风险识别框架
- 持仓心态框架
- 总结记忆点

### 改写后不应出现的内容
- 4060 / 4100 这类具体点位
- 七成仓 / 八成仓这类仓位比例
- 加仓 / 减仓 / 低吸 / 高抛
- 明天高开 / 低开 / 震荡 / 变盘
- 具体板块机会建议
- 具体个股应对口径
- 点关注 / 人气票 / 扣1扣2

### 输出气质
- 更像知识直播
- 更像理性拆解型老师
- 更适合长期复用
- 更像讲方法，不像讲操作
- 有直播感，但整体克制

### 成功判断标准
- 策略属性明显下降
- 知识框架明显增强
- 不再直接给明天怎么做
- 老师可以直接用于知识型直播
- 结尾可以压成几个可带走的框架

### 备注
当原稿同时存在“策略属性重、文章腔重、互动低质、风格不贴老师”四类问题时，优先按本模式处理：
先降级为知识直播框架稿，再做口播优化和老师贴脸调整。
