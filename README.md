一、这个 Skill 是干什么的
make-stock-live 是一个 股票知识直播内容中控 Skill。
它不是普通写稿工具。
它的作用是帮你把一场股票知识直播，按顺序拆成：
- 讲什么
- 怎么铺整场
- 每一段怎么讲
- 最后具体说什么
也就是：
主题 → 大框架 → 小框架 → 逐字稿
如果你有同行稿，它还能帮你：
- 拆同行逻辑
- 提炼框架
- 转成你们自己的可用结构
如果你有旧稿，它还能帮你：
- 找问题
- 改得更像直播
- 变得更抓人，但不踩合规红线

---
二、它最适合解决哪些问题
你们平时直播前最常见的问题，一般就是这几类：
1. 不知道今天讲什么
老师脑子里没题，开播前不知道讲哪个方向。
2. 有主题，但不会铺90分钟
知道要讲“反弹和反转”，但不知道整场怎么安排。
3. 有整场框架，但某一段不会讲
比如第4段要讲判断框架，但不知道怎么展开。
4. 知道这段讲什么，但不会说
这时候就需要逐字稿。
5. 有同行稿，想提炼逻辑
不是照抄，而是拆结构。
6. 有原稿，但不够像直播
太书面、太空、不抓人，就让它优化。

---
三、这个 Skill 的 6 个核心模式
1. build
用途：生成整场 90 分钟直播大框架
适合：
- 老师不知道怎么铺整场
- 只有主题，没有结构
输出：
- 整场分几段
- 每段讲多久
- 每段的目标
- 每段建议用什么小框架

---
2. outline
用途：拆某一段的小框架
适合：
- 已经有整场框架
- 但某一段不会展开讲
输出：
- 本段目标
- 本段讲解顺序
- 核心讲点
- 钩子方向
- 承接方向
- 记忆点方向

---
3. script
用途：生成可直接口播的逐字稿
适合：
- 已经有主题 / 大框架 / 小框架
- 需要直接拿去讲
输出：
- 稳健版
- 强钩子版
- 主播语气提示
- 可替换开头
- 可替换收尾

---
4. reverse
用途：拆同行逐字稿，提炼结构
适合：
- 你有同行稿
- 想看对方整场怎么讲
- 想提炼框架而不是抄表达
输出：
- 大框架提炼
- 小框架提炼
- 留人点分析
- 含金量分析
- 可迁移版新结构

---
5. polish
用途：优化已有稿件
适合：
- 原稿太书面
- 不抓人
- 太空
- 不像直播
输出：
- 原稿主要问题
- 修改优先级
- 优化版A
- 优化版B

---
6. review
用途：给稿件评分和诊断
适合：
- 想看这稿子能不能播
- 想知道最大问题在哪
输出：
- 当前稿件诊断
- 质量评分
- 修改建议
- 是否可直接上播

---
四、最标准的使用顺序
平时最稳的用法，不是乱跳，而是按下面这个顺序：
场景一：老师什么都没有
先走：
build → outline → script
也就是：
1. 先定整场大框架
2. 再拆某一段小框架
3. 再生成逐字稿

---
场景二：老师有主题，但不会讲
先走：
build → outline → script
和上面一样，只是你省掉选题这一步。

---
场景三：老师已经有整场框架
直接走：
outline → script

---
场景四：有同行稿
走：
reverse → build / outline → script
先拆同行结构，再转成自己的框架，再出稿。

---
场景五：有旧稿，要优化
走：
polish → review

---
场景六：只想看看稿子行不行
直接走：
review

六、平时怎么问，效果最好
不要只说：
帮我写个直播稿
这样太模糊。
最好每次都尽量补上这几个信息：
- 主题
- 时长
- 老师风格
- 受众
- 直播目标
例如：
/make-stock-live 主题是“为什么很多人总把反弹当反转”，做一场90分钟股票知识直播，老师偏理性拆解型，受众是普通散户，目标是留人和建信任，先给我整场大框架
这样出来的结果会稳很多。

七、它最适合谁用
这个 Skill 不是只给一个人用的。
老师
适合看：
- 讲什么
- 怎么讲
- 直接拿稿讲
运营 / 编导
适合用来：
- 定选题
- 搭整场框架
- 拆某一段逻辑
- 改稿
内容负责人 / 操盘手
适合用来：
- 看结构合不合理
- 看老师讲法稳不稳
- 看内容有没有复用价值

---
八、怎么判断它给的结果好不好
你拿到结果后，主要看这 5 件事：
1. 有没有逻辑
是不是一段一段有顺序，不是乱讲。
2. 有没有直播感
是不是像老师在说，不是像文章。
3. 有没有含金量
是不是有误区、框架、案例、记忆点。
4. 能不能直接拿去讲
不是只会看，不会讲。
5. 有没有踩边界
是不是还保持在股票知识直播，而不是诱导交易。

---
九、如果觉得结果不够好怎么办
不用慌，直接继续往下问。
情况1：整场太空
你就说：
/make-stock-live 这个大框架太空了，帮我把第3段和第4段拆得更具体一点
情况2：逐字稿太像文章
你就说：
/make-stock-live 这段太像文章了，改得更像直播口播，更自然一点
情况3：不够抓人
你就说：
/make-stock-live 开头抓力不够，帮我把前10秒改强一点，但不要踩合规红线
情况4：不像某个老师
你就说：
/make-stock-live 这段还不够像我们老师，语气更理性一点，少一点太满的表达
也就是说，它不是一次出完就不能动。
你可以继续“追问式调整”。

---
十、最简单的一句话记住
不知道讲什么，就让它出主题和大框架
不知道这一段怎么讲，就让它出小框架
不知道具体怎么说，就让它出逐字稿
不知道哪里不好，就让它帮你改稿和评分

---
十一、推荐你们团队内部固定用法
如果你要给团队统一用，我建议就固定成这套：
第一步
先出整场大框架
第二步
只挑最关键的 2 到 3 段拆小框架
第三步
再只生成关键段逐字稿
第四步
最后 review 一遍
这样最省时间，也最稳。
